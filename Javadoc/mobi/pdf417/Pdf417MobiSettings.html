      • setAllowMultipleScanResultsOnSingleImage

        public void setAllowMultipleScanResultsOnSingleImage(boolean multiScan)
        Sets whether or not outputting of multiple scan results from same image is allowed. If that is true, it is possible to return multiple recognition results from same image (e.g. both PDF417 and QR code from same image can be decoded). By default, this option is true. If you turn it off, then list of Pdf417MobiScanData obtained via intent will contain at most 1 element. The upside of turning that option off is the speed - if you enable lots of recognizers, as soon as the first recognizer succeeds in reading the barcode or slip, function will return and other recognizers will not get a chance to analyze the image. The downside is that you are then unable to obtain multiple results from single image.
