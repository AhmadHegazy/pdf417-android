      • setNewTarget

        public void setNewTarget(net.photopay.geometry.Quadrilateral objPosition,
                int detectionStatus)
        This method is called to notify viewfinder that object has been detected so that it can draw itself around that object. Corner points of detected slip are presented via Quadrilateral object. Quadrilateral's corner point coordinates are already prepared for drawing on any fixed view. If you want to draw them on rotatable view, you need to rotate their coordinates accordingly. Default implementation does nothing. This method is called from background thread. Do not perform UI operations directly in this method, instead use Handler.
        Parameters:
        detectionStatus - The status of detection as defined in RecognizerDelegate

      • setPointSet

        public void setPointSet(net.photopay.geometry.PointSet pointSet)
        This method is called to notify viewfinder that object has been detected so that it can draw points of interest for this object. Default implementation does nothing. This method is called from background thread. Do not perform UI operations directly in this method, instead use Handler. Point coordinates are already adjusted for drawing on fixed view. In order to draw them in rotatable view, you have to apply appropriate rotation transformation to point coordinates.
        Parameters:
        pointSet - Array of coordinates for points of interest. Coordinates are packed in following way: x1,y1,x2,y2,...

      • getRotatableView

        public android.view.View getRotatableView()
        This method is called when adding viewfinder on top of camera surface. This method should return view that will be overlayed on camera. The view returned in this method will be automatically rotated when device orientation changes. If you do not want to rotate your UI, simply return null here and return your fixed UI in method getFixedView. Of course, you can return both fixed and rotatable view - in this case, rotatable view will be overlayed on top of fixed view which will be overlayed on top of camera surface. Default implementation returns null.
        Returns:
        View that should be overlayed on camera.

        public boolean shouldAnimateRotation()
        You should return true in this method if you want your rotatable UI to be animated while performing a rotation. Default value is false. Note that on some devices animating the rotation sometimes causes draw corruption - this is the reason why is the default return value false.
        Returns:
        true if rotatable view should be animated while rotating, otherwise false.
